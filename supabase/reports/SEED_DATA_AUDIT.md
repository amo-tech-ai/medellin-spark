## üö® Red Flags (Fix These First)

### 1. ‚ùå Direct Writes to `auth.users`

**Problem**: Lines 34-169
```sql
DELETE FROM auth.users WHERE email LIKE '%@medellin-spark.local';

INSERT INTO auth.users (
  id,
  instance_id,
  email,
  encrypted_password,  -- ‚ö†Ô∏è PROBLEM
  email_confirmed_at,
  created_at,
  updated_at,
  raw_app_meta_data,
  raw_user_meta_data,
  is_super_admin,      -- ‚ö†Ô∏è DEPRECATED
  role
) VALUES (...)
```

**Why It's Wrong**:
- ‚ùå Supabase Cloud **blocks** inserts to `auth.users`
- ‚ùå Newer GoTrue (v2.99.0+) stores passwords in `auth.identities`, NOT `auth.users.encrypted_password`
- ‚ùå Columns like `instance_id`, `is_super_admin`, `role` **differ by version**
- ‚ùå Schema drift: GoTrue can change internal structure without notice
- ‚ùå `DELETE FROM auth.users` will be rejected on Cloud

**Impact**: 
- üî¥ **CRITICAL**: Will fail on Supabase Cloud (100% guaranteed)
- üü° **HIGH**: May fail on local Supabase with newer GoTrue versions
- üü° **HIGH**: Brittle to GoTrue schema changes

**Fix**:
```bash
# Use Auth Admin API/CLI instead
supabase auth admin create-user \
  --email sofia.martinez@medellin-spark.local \
  --password password123 \
  --email-confirm
```

### 2. ‚ùå Password Seeding via `crypt()`

**Problem**: Line 53, 80, 107, 134, 161
```sql
encrypted_password = crypt('password123', gen_salt('bf'))
```

**Why It's Wrong**:
- ‚ùå Depends on `pgcrypto` extension (not guaranteed to exist)
- ‚ùå Relies on specific GoTrue password hashing internals
- ‚ùå GoTrue v2+ uses **Argon2** or **bcrypt via auth.identities**, NOT `auth.users.encrypted_password`
- ‚ùå Format/salt may not match what GoTrue expects

**Impact**:
- üî¥ **CRITICAL**: Users created this way may not be able to log in
- üü° **HIGH**: Missing `pgcrypto` will cause SQL error

**Fix**:
```sql
-- Add to migrations (if you must use crypt elsewhere)
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- BUT for auth users, use CLI instead:
supabase auth admin create-user --password password123
```

### 3. ‚ùå Assumes Auth Schema Mutability

**Problem**: Line 34
```sql

**Why It's Wrong**:

**Impact**:
- üî¥ **CRITICAL**: Fails on Cloud
- üü° **MEDIUM**: May leave orphaned data in `auth.identities`, `auth.sessions`, etc.

**Fix**:

```
